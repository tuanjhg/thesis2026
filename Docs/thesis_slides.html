Introduction
Motivation & Problem Statement
UETVNU

Bối cảnh

  • Tấn công DDoS trong Data Center Network (DCN) tăng cả về quy mô và độ tinh vi (multi-vector, low-and-slow).
  • Phòng thủ dựa trên ngưỡng cứng (rule-based) phản ứng chậm và dễ bị bỏ lọt các mẫu tấn công mới.
  • NFV + SDN (ONAP) cho phép điều phối động chuỗi dịch vụ VNF (SFC) nhưng cần tín hiệu AI dự báo chính xác.

Câu hỏi nghiên cứu

  • RQ1: Làm sao kết hợp AI (phát hiện + dự báo) vào vòng kín ONAP để giảm thiểu DDoS chủ động?
  • RQ2: Lợi thế về lead-time của nhánh proactive (T2) so với reactive (T3) là bao nhiêu?
  • RQ3: Kiến trúc có mở rộng được sang topology DCN chuẩn (fat-tree) không?

Background
NFV · ONAP · ML for DDoS
UETVNU

NFV & SFC

Network Function Virtualization ảo hoá các hàm mạng (firewall, rate-limiter, scrubber) thành VNF. Service Function Chaining (SFC) định tuyến lưu lượng qua chuỗi VNF theo chính sách.

ONAP Closed-Loop

SO (Service Orchestrator) — CLAMP (Closed-Loop Automation) — Policy Framework — DMaaP (bus) tạo vòng kín: telemetry → phân tích → quyết định → tác động.

XGBoost 7-class

Phân loại: Normal, UDP_Flood, SYN_Flood, DNS_Amp, NTP_Amp, MSSQL, Benign-Misclass. AUC đạt 0.9999 trên CICDDoS2019.

Transformer + LSTM Forecaster

Dự báo xác suất tấn công ở 4 horizon (30/60/90/120 s) → sinh tín hiệu proactive pre-position khi P(t+30s) > 0.5.

Limitations & Future Work
Hạn chế & hướng phát triển
UETVNU

Hạn chế

  • ONAP SO/CLAMP hiện dùng Docker stub, chưa tích hợp ONAP OOM thật.
  • Fat-tree k=4 = 16 host — DCN thực tế cần k≥8 (128 host).
  • Forecast chỉ 4 horizon rời rạc (30/60/90/120 s).
  • Adversarial training mới chỉ FGSM; chưa đánh giá PGD/BIM.

Hướng phát triển

  • Tích hợp ONAP OOM thật cho ≥ 1 kịch bản end-to-end.
  • Autoregressive multi-step forecast (seq2seq).
  • Mở rộng sang fat-tree k=6/k=8 để đo knee-point orchestrator.
  • Adaptive attacker threat model (S9 low-and-slow).
  • Reinforcement Learning controller thay tier-mapper tĩnh.

Conclusion
Đóng góp chính
UETVNU

Các đóng góp

  • C1 · Kiến trúc 4 tầng AI-augmented ONAP — closed-loop thật sự với 3 actuator (SO, CLAMP, SFC).
  • C2 · Proactive pre-position — Transformer+LSTM 4-horizon kích hoạt T2 trước T3 reactive, lead-time ~5.5 s/ca.
  • C3 · Đánh giá định lượng S1–S8 — 8/8 PASS, AUC 0.9999, so sánh baseline threshold.
  • C4 · Testbed mở — Mininet PAD + fat-tree k=4, toàn bộ code & script reproduce được cung cấp.
  • C5 · OOD robustness — HTTP/ICMP ngoài tập huấn luyện vẫn giữ T0–T1, không over-escalate.

Khẳng định trọng tâm

Nhánh proactive T2 đạt p50 ≈ 505 ms so với nhánh reactive T3 ≈ 6006 ms — khẳng định tính khả thi của AI-augmented NFV orchestration cho DDoS trong DCN.
